## 🚀 最佳方案：半自动化部署

### 第一步：你创建项目（只需 1 分钟）

1. 访问：https://railway.app/dashboard
2. 点击 **"New Project"** → **"Deploy from GitHub repo"**
3. 选择仓库：`leizhuofan2005/dfs-mazesolver`
4. Railway 会自动创建项目并开始部署

### 第二步：我通过 MCP 完成剩余工作

创建项目后，告诉我，我会自动：

1. ✅ **配置后端服务**
   - 设置构建命令：`pip install -r backend/requirements.txt`
   - 设置启动命令：`cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT`
   - 获取后端 URL

2. ✅ **创建并配置前端服务**
   - 从同一个 GitHub 仓库创建前端服务
   - 设置构建命令：`npm install && npm run build`
   - 设置启动命令：`npx serve -s dist -l $PORT`
   - 设置环境变量 `VITE_API_URL`（指向后端 URL）

3. ✅ **配置 CORS**
   - 在后端设置 `ALLOWED_ORIGINS` 环境变量

4. ✅ **配置域名**
   - 为前后端服务生成域名

## 💡 为什么需要这一步？

Railway 的项目创建需要通过 Web 界面进行 GitHub 授权和仓库选择，这是安全机制的一部分。一旦项目创建完成，所有后续配置都可以通过 MCP 自动化完成。
