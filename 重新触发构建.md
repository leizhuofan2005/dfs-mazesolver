# 重新触发 GitHub Actions 构建

## 方法 1: 手动触发（推荐）

1. **访问 GitHub Actions 页面**
   - https://github.com/leizhuofan2005/dfs-mazesolver/actions

2. **点击左侧的 workflow**："Build and Push to ACR"

3. **点击 "Run workflow" 按钮**（右上角）

4. **选择分支**：`main`

5. **点击绿色的 "Run workflow" 按钮**

## 方法 2: 推送空提交

如果方法 1 不可用，运行：

```powershell
git commit --allow-empty -m "Retry build"
git push
```

## 方法 3: 修改任意文件后推送

```powershell
# 创建一个临时文件
echo "# Build retry" > .build-retry
git add .build-retry
git commit -m "Retry build"
git push
```

## 检查构建状态

构建触发后：

1. **查看构建状态**
   - https://github.com/leizhuofan2005/dfs-mazesolver/actions

2. **检查登录步骤**
   - 如果 "Login to Alibaba Cloud ACR" 成功 → Secrets 配置正确 ✅
   - 如果仍然失败 → 检查 Secret 名称和值

3. **等待构建完成**
   - 预计时间：5-10 分钟

## 如果登录成功但构建失败

检查：
- Dockerfile 是否有错误
- 构建日志中的具体错误信息
- 依赖项是否正确

## 如果登录仍然失败

1. **再次检查 Secret 名称**（必须完全一致）：
   - `ALIYUN_USERNAME`
   - `ALIYUN_ACR_PASSWORD`

2. **检查 Secret 值**：
   - 不为空
   - 没有多余空格
   - 使用正确的值（AccessKey 或账号密码）

3. **参考**：`修复GitHub Secrets.md`

---

**推荐使用方法 1（手动触发），最简单直接！**
